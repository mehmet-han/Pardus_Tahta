# Fatih Client - Important Updates

## What's New

This update includes **4 critical bug fixes** and **1 new feature**:

### Bug Fixes ✅

1. **USB Unlock Loop** - Fixed infinite lock/unlock cycle when USB key used during scheduled lock period
2. **Schedule Parsing Error** - Fixed `AttributeError: 'list' object has no attribute 'get'` crash
3. **SSL Warning Spam** - Suppressed repetitive InsecureRequestWarning messages
4. **Lock Screen Not Showing** - Fixed issue where lock screen didn't appear after reboot

See [FIXES.md](FIXES.md) for detailed technical information.

### New Feature 🎉

**Pre-Login Lock Screen** - Configure system to show Fatih lock BEFORE user login
- Prevents any user from logging in until admin unlocks
- Perfect for schools with multiple user accounts
- See [PRE-LOGIN-SETUP.md](PRE-LOGIN-SETUP.md) for installation guide

## Quick Deployment Guide

### For Current Bug Fixes (No Pre-Login)

On your **development machine**:

```bash
cd /home/cappittall/Documents/Fatih_Projesi
git add .
git commit -m "Fix USB loop, schedule parsing, SSL warnings, and lock screen visibility"
git push
```

On the **remote machine** (via SSH):

```bash
cd ~/Fatih_Projesi
git pull

# Stop current client
pkill -f client.py

# Copy updated files
sudo cp fatih_projesi_python/client/client.py /opt/fatih-client/

# Watchdog will auto-restart in ~10 seconds
# Or restart manually:
sudo -u etapadmin /opt/fatih-client/launch.sh --foreground
```

**Test:**
- Lock screen should appear immediately
- USB unlock should work without looping
- No SSL warnings in logs

### For Pre-Login Lock Screen (Optional)

This is an **additional feature** that makes Fatih lock appear BEFORE the Pardus login screen.

**⚠️ Warning:** This changes system login behavior. Test on a non-production machine first!

On the **remote machine**:

```bash
cd ~/Fatih_Projesi/fatih_projesi_python/client

# Run pre-login installer
sudo ./install-prelogin.sh

# Follow the prompts, then reboot
sudo reboot
```

**After reboot:**
1. Fatih lock screen appears immediately
2. Enter admin password (default: `803580`)
3. GDM3 login appears with your 3 users (ogretmen, yetkili, ogrenci)
4. Users can now log in normally

See [PRE-LOGIN-SETUP.md](PRE-LOGIN-SETUP.md) for complete documentation.

## Troubleshooting

### Client not starting after update

```bash
# Check logs
tail -50 /opt/fatih-client/logs/client.err.log
tail -50 /opt/fatih-client/logs/watchdog.log

# Check if Python script has syntax errors
/opt/fatih-client/venv/bin/python /opt/fatih-client/client.py --test
```

### Lock screen not visible

```bash
# Check process
ps aux | grep client.py

# Check display environment
echo $DISPLAY
echo $XAUTHORITY

# Restart with foreground mode for debugging
sudo -u etapadmin /opt/fatih-client/launch.sh --foreground
```

### USB loop still occurring

```bash
# Check if you have the latest code
grep "Keep manual_override active" /opt/fatih-client/client.py

# Should return: "# Keep manual_override active until the next scheduled unlock period"
```

### Pre-login not working

```bash
# Verify GDM3 config
cat /etc/gdm3/custom.conf | grep AutomaticLogin

# Check kiosk logs
sudo cat /opt/fatih-client/logs/kiosk.log

# Verify fatih-kiosk user
id fatih-kiosk
```

## Rollback Instructions

### Rollback to Previous Version

```bash
cd ~/Fatih_Projesi
git log --oneline  # Find previous commit hash
git checkout <previous-commit-hash>
sudo cp fatih_projesi_python/client/client.py /opt/fatih-client/
pkill -f client.py
```

### Disable Pre-Login Lock

```bash
sudo sed -i 's/AutomaticLoginEnable = true/AutomaticLoginEnable = false/' /etc/gdm3/custom.conf
sudo reboot
```
